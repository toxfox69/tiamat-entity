Frequently Asked Questions

How does drift detection work?

You register a model, send 20+ baseline samples, then check new batches against that baseline. We run the appropriate statistical test — PSI for numeric outputs, cosine distance for embeddings, KL divergence for probability distributions, or text statistics for LLM outputs — and return a score with an alert flag if the threshold is exceeded.

What counts as a "drift check"?

One POST /drift/check call with any number of samples (up to 10,000) counts as a single check. Checks are tracked per IP per day on Free, and unlimited on paid plans.

Can I set my own alert thresholds?

Pro and Enterprise plans can pass a "threshold" key in the model's config at registration time. Free tier uses our defaults: 0.25 for PSI, 0.15 for cosine, 0.20 for entropy and text.

How do webhook alerts work?

Pass a "webhook_url" in your model config at registration. When drift exceeds threshold, we POST a JSON payload with model ID, score, method, and timestamp to your endpoint within seconds of detection. Test delivery with POST /drift/alert/test.

Can I monitor LLM / generative AI outputs?

Yes. Use model_type: "text" to track response length and vocabulary diversity shifts. Use model_type: "probability" for token probability distributions, or model_type: "embedding" for semantic embeddings from a retrieval layer.

What payment methods do you accept?

Pro plans can be paid via credit card or USDC on Base mainnet via our x402 payment protocol. Enterprise invoicing available. Pay-per-use (no subscription) is also supported at $0.01 USDC per check beyond the free tier.

Is there an SDK?

Yes — a Python SDK is available. It wraps all endpoints with retries, batch helpers, and Slack/webhook configuration. See the docs or install: pip install tiamat-drift (coming soon).

Can I self-host?

Enterprise plans include on-premise and VPC deployment options. The core drift engine is pure Python + NumPy with SQLite — no external dependencies. Contact sales to discuss your deployment requirements.

Start monitoring in under 5 minutes

Two API calls to register and baseline your model. No infrastructure to set up.

Try free — no card needed Book a demo 
# Register, baseline, and detect drift in 3 calls
# Step 1: Register your model
curl -X POST https://tiamat.live/drift/register \
  -H "Content-Type: application/json" \
  -d '{"name":"revenue-classifier","model_type":"numeric"}'
# → {"model_id": 42, ...}

# Step 2: Set baseline (production scores from last 30 days)
curl -X POST https://tiamat.live/drift/baseline \
  -d '{"model_id":42,"samples":[0.92,0.88,0.94,0.87,...]}'

# Step 3: Check today's batch
curl -X POST https://tiamat.live/drift/check \
  -d '{"model_id":42,"samples":[0.41,0.38,0.55,...]}'
# → {"score": 0.34, "alert": true, "method": "psi", "threshold": 0.25}
