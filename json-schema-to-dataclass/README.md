# schema_to_dataclass

Convert JSON Schema files to Python `@dataclass` modules with full type hints,
`Optional` fields, nested class generation, and validation docstrings.

**Zero runtime dependencies** — stdlib only (Python 3.10+).

---

## Installation

```bash
pip install .
# or directly:
pip install -e .
```

After install the `schema_to_dataclass` command is available globally.

---

## Quick Start

```bash
# Write output to User.py
schema_to_dataclass schema.json -o User.py

# Auto-name class from schema title
schema_to_dataclass schema.json

# Print to stdout
schema_to_dataclass schema.json --stdout

# Override class name
schema_to_dataclass schema.json -c Order -o Order.py

# Read from stdin
cat schema.json | schema_to_dataclass - --stdout -c Event
```

## Example

**Input (`user.json`)**

```json
{
  "title": "User",
  "type": "object",
  "description": "Represents a registered user.",
  "properties": {
    "id":       { "type": "integer" },
    "name":     { "type": "string" },
    "email":    { "type": "string", "format": "email" },
    "age":      { "type": "integer", "minimum": 0, "maximum": 150 },
    "score":    { "type": "number" },
    "active":   { "type": "boolean" },
    "tags":     { "type": "array", "items": { "type": "string" } },
    "address": {
      "type": "object",
      "properties": {
        "street": { "type": "string" },
        "city":   { "type": "string" },
        "zip":    { "type": "string" }
      },
      "required": ["street", "city"]
    }
  },
  "required": ["id", "name", "email"]
}
```

**Output (`User.py`)**

```python
"""Auto-generated by schema_to_dataclass — do not edit manually."""
from __future__ import annotations

from dataclasses import dataclass, field
from typing import List, Optional


@dataclass
class Address:
    """
    Attributes:
        street: street
        city: city
        zip: optional; zip
    """
    street: str
    city: str
    zip: Optional[str] = None


@dataclass
class User:
    """
    Represents a registered user.

    Attributes:
        id: id
        name: name
        email: RFC 5322 email address
        age: optional; minimum=0; maximum=150
        score: optional; score
        active: optional; active
        tags: optional; tags
        address: optional; address
    """
    id: int
    name: str
    email: str
    age: Optional[int] = None
    score: Optional[float] = None
    active: Optional[bool] = None
    tags: Optional[List[str]] = None
    address: Optional[Address] = None
```

## Supported JSON Schema Features

| Feature | Supported |
|---------|-----------|
| Scalar types: `string`, `integer`, `number`, `boolean`, `null` | ✅ |
| `required` → required fields (no default) | ✅ |
| Non-required → `Optional[T] = None` | ✅ |
| Nested `object` → new `@dataclass` | ✅ |
| `array` with `items` → `List[T]` | ✅ |
| `anyOf` / `oneOf` → `Union[...]` or `Optional[...]` | ✅ |
| `allOf` (simplified merge) | ✅ |
| `$ref` (local definitions) | ✅ |
| `definitions` / `$defs` | ✅ |
| `default` values | ✅ |
| Format hints in docstrings | ✅ |
| Constraint annotations (min/max/pattern/enum) | ✅ |
| Free-form objects (no properties) → `Dict[str, Any]` | ✅ |
| Schema `title` → class name | ✅ |
| Type arrays e.g. `["string", "null"]` | ✅ |
| Python keyword field names (escaped) | ✅ |

## CLI Reference

```
usage: schema_to_dataclass [-h] [-o FILE] [-c NAME] [--stdout] [SCHEMA]

positional arguments:
  SCHEMA          JSON Schema file path, or '-' for stdin (default: stdin)

options:
  -h, --help      show this help message and exit
  -o FILE         Output .py file (default: <ClassName>.py)
  -c NAME         Root class name (default: from schema title or filename)
  --stdout        Print to stdout instead of writing a file
```
